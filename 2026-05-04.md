## 오늘 끝난 것 (참고용 — 다시 손댈 필요 없음)

### 1. CSS MIME 에러 (`/css/tailwindcss` 404 + `application/json`)
- 원인: `partials/sidebar.html`이 `input.css`를 직접 link → 그 안의 `@import "tailwindcss"`를 브라우저가 fetch 시도
- 해결: `partials/sidebar.html`의 `<link rel="stylesheet" th:href="@{/css/input.css}" />` 라인 삭제
- 주의: input.css는 빌드 소스파일이지 브라우저용이 아님. 항상 `output.css`만 link

### 2. 사이드바 로고 크기 페이지마다 다름
- 원인: `output.css`에 `.h-12` 클래스가 빌드되어 있지 않음. `cdn.tailwindcss.com` 스크립트가 있는 페이지만 런타임에 보충됨. index.html에는 그 CDN이 없어 본래 이미지 크기로 큼
- 해결: Tailwind 재빌드 완료
  - `npm install` 한 번 (702 packages)
  - `input.css`의 `@apply ... !important` → v4 문법 `bg-indigo-600! text-white!`로 수정 (input.css 195행 근처)
  - `npx @tailwindcss/cli -i ./src/main/resources/static/css/input.css -o ./src/main/resources/static/css/output.css`
- 다음에 input.css 수정하면 같은 빌드 명령 다시 돌려야 함

### 3. waitingList 빈 화면
- CSS 문제 아니었음. DB의 `tbl_member`에 `status='WAIT'`인 회원이 0명이라 `${members}` 빈 리스트 → 카드 안 그려짐. 정상 동작.

## 진행 중인 작업: 직급 관리(managingPosition)에 관리자 체크박스 + AJAX

### 배경/현재 상태

대상 파일 3개:
- `src/main/resources/templates/admin/managingPosition.html`
- `src/main/resources/static/js/admin/position.js`
- `src/main/java/com/team/intranet/controller/api/PositionApiController.java`

확인된 현재 상태:
- HTML 폼 상단에 `isAdminCheck` 체크박스는 있지만, 목록 행에는 관리자 여부 컬럼 **없음**
- `position.js`에 `createPosition` 함수가 **두 번 정의**되어 두 번째가 첫 번째를 덮어씀. 두 번째는 fetch가 **주석 처리**되어 있어 현재 직급 생성이 서버로 안 가는 상태
- `savePosition`에 죽은 코드 `posCode` 참조 있음
- 컨트롤러 3개 엔드포인트(`create`/`update/{id}`/`delete/{id}`)는 `PositionDto` 받게 잘 만들어져 있음 → DTO만 바뀌면 컨트롤러 수정 거의 없음
- 컨트롤러는 응답으로 단순 텍스트만 보내는데 js는 `htmlChunk`를 `innerHTML`에 박음 → 정합성 문제 있음 (9단계에서 정리)

### 데이터 모델 결정 (이미 정함)

- `Position` 엔티티에 `Role role` 필드 추가 (Member의 Role enum 그대로 재사용 — `ADMIN`/`USER`/`MASTER`)
- `Member.role`은 **유지** (지우지 말 것)
  - 이유 1: position이 null인 회원 케이스 존재 (WAIT/REJECT/LEAVE 상태)
  - 이유 2: `MASTER`는 직급과 독립
  - 이유 3: 권한 체크 성능. MemberSession에 role만 담으면 join 불필요
- `positionLevel`은 직급 정렬용으로 쓰이고 있어 **유지**
- 관계: `Position.role` = 마스터 데이터(이 직급은 관리자 직급이다 라는 규칙), `Member.role` = 거기서 파생되는 결과값
- 체크박스 UI는 boolean으로 받고 서버에서 `Role.ADMIN`/`Role.USER`로 변환 (DTO는 `boolean isAdmin`로 받는 게 깔끔)

### 단계 가이드

권장 순서: **0 → 1 → 2 → 3 → 4 → 5 → 6 → 7 → 9 → 8 → 10**

#### 0단계 — 사용처 재확인 (완료)
- `positionLevel` 정렬용으로 사용 중 → 유지
- `Role` enum 위치: `com.team.intranet.enums.member.Role`

#### 1단계 — `Position` 엔티티 변경
파일: `src/main/java/com/team/intranet/entity/Position.java`
- `private int positionLevel` 유지
- `private Role role` 추가 (`@Enumerated(EnumType.STRING)`, `@Column(name="role")`)
- `createPosition` 정적 메서드 시그니처에 role 인자 추가 (또는 기본값 `Role.USER`로 셋)
- `@AllArgsConstructor`라 생성자 자동 갱신 — 이걸 호출하는 곳(특히 `PositionDto.toEntity`) 다 수정 필요

#### 2단계 — `PositionDto` 변경
파일: `src/main/java/com/team/intranet/dto/PositionDto.java`
- `positionLevel` 유지
- `boolean isAdmin` 필드 추가 (UI 친화)
- `toEntity()` 갱신: `new Position(null, positionName, company, positionLevel, isAdmin ? Role.ADMIN : Role.USER)`

#### 3단계 — DB 컬럼
- `tbl_position`에 `role VARCHAR2(20)` 컬럼 추가
- 기존 row의 role 기본값을 `'USER'`로 update
- `application.yml`의 `ddl-auto` 정책 확인. `update`면 자동, 아니면 ALTER 직접 실행

#### 4단계 — `position.js` 정리: createPosition 중복 제거 + 체크박스 활성
파일: `src/main/resources/static/js/admin/position.js`
- 두 번째 `createPosition`(42~69행) 통째로 삭제
- 첫 번째 `createPosition`(8~40행)에 다음 추가:
  - `document.getElementById('isAdminCheck').checked` 값 읽어 `data.isAdmin`에 포함
  - 헤더에 CSRF 토큰 추가 (`getCsrfToken()` 사용 — 다른 함수와 일관)

#### 5단계 — managingPosition.html 목록 행에 체크박스 컬럼 추가
파일: `src/main/resources/templates/admin/managingPosition.html` (74행 근처 `<div th:each="pos : ${positions}">`)
- 각 행에 체크박스 추가
- 초기 상태: `th:checked="${pos.role != null && pos.role.name() == 'ADMIN'}"` (또는 dto/entity에 `isAdmin()` getter 두기)
- 일반 모드: `disabled` 속성 부여
- 수정 모드 진입 시 활성화 (6단계 토글 로직과 함께)
- 행마다 ID 부여 권장: `pos-isAdmin-${pos.positionId}`

#### 6단계 — `position.js`의 toggleEditMode/savePosition 보완
- `toggleEditMode`: 위 체크박스의 `disabled` 토글 추가
  - 수정 모드 진입 시 원래 값을 `data-original` 속성에 저장
  - 취소 시 원래 값으로 복원
- `savePosition`: 체크박스 값 읽어 `isAdmin`을 요청 body에 포함
- 죽은 코드 `posCode` 참조 라인 정리

#### 7단계 — Member.role 동기화 정책 (가장 중요)
세 시점에 동기화:
1. **가입 승인 시점**: `Member.accept(dept, position)` 메서드 안에 `this.role = position.getRole()` 추가 (단, position null 가드 필요)
2. **회원 직급 변경 시점**: `Member.updateInfo`에서 position이 변경되면 `this.role = position.getRole()` 동기화
3. **직급 자체의 role 변경 시점 (체크박스 토글로 직급의 role이 바뀔 때)**: `PositionService.updatePosition`에서 그 직급을 가진 회원들의 role 일괄 update
   - 쿼리 조건에 **`AND role <> 'MASTER'` 가드 필수** (마스터 권한자는 절대 덮어쓰면 안 됨)

#### 8단계 — 컨트롤러
파일: `src/main/java/com/team/intranet/controller/api/PositionApiController.java`
- DTO 자동 매핑이라 변경 거의 없음
- 9단계 정합성 결정에 따라 응답 형태만 조정

#### 9단계 — 응답 정합성 정리 (현재 깨져있음)
컨트롤러는 텍스트 메시지만 보내는데 js는 `innerHTML`에 박는 문제. 셋 중 택1:
- (a) 컨트롤러가 Thymeleaf fragment 반환 (`return "admin/managingPosition :: positionList"`) — 현재 js 그대로 사용 가능. 추천
- (b) JSON으로 갱신된 목록 반환, js에서 row를 다시 그림
- (c) 단순화: 응답 후 `location.reload()`

#### 10단계 — 검증
- 직급 생성 시 체크 → DB의 role = ADMIN 확인

- 직급 수정으로 체크박스 토글 → DB 갱신 + 그 직급 가진 회원들 role 일괄 갱신 확인 (MASTER는 그대로인지)
- 새로고침 후 체크 상태 유지 확인
